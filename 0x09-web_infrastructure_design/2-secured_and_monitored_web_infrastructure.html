üîí Security & Monitoring Elements:

  • 3 Firewalls: Filter traffic, block unauthorized access, protect each server
  • SSL Certificate: Enables HTTPS encryption for secure communication
  • HTTPS Purpose: Encrypts data in transit, prevents eavesdropping and tampering
  • 3 Monitoring Clients: Collect system metrics, logs, and performance data
  • Monitoring Data Collection: Agents send logs/metrics to Sumologic for analysis
  • QPS Monitoring: Configure web server access logs to track queries per second

üìä How Monitoring Works:

  • Data Collectors: Installed on each server to gather metrics
  • Metrics Collected: CPU usage, memory, disk space, network traffic
  • Log Collection: Application logs, error logs, access logs
  • Alerting: Notifications when thresholds exceeded or errors occur
  • Web Server QPS: Parse Nginx access logs to count requests per second

‚ö†Ô∏è Remaining Infrastructure Issues:

  • SSL Termination at Load Balancer: Traffic between LB and servers is unencrypted
  • Single MySQL Write Point: Only one server can handle database writes (bottleneck)
  • Identical Server Components: All servers run web+app+db (inefficient resource usage)
  • No Database Redundancy: Database failure affects all servers
  • Resource Waste: Each server doing everything instead of specialized roles

üî• Firewall Functions:

  • External Firewall: First line of defense, blocks malicious traffic
  • Server Firewalls: Protect individual servers, allow only necessary ports
  • Traffic Filtering: Block suspicious IPs, limit connection rates
  • Port Security: Only allow HTTP(80), HTTPS(443), SSH(22) as needed
