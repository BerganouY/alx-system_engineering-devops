üîç Why We Added Each Element:

  • Load Balancer (HAProxy): Distributes traffic across multiple servers for reliability
  • 2 Additional Servers: Eliminates single server SPOF, handles more traffic
  • Distribution Algorithm: Round Robin - sends requests to servers in rotation
  • Active-Active Setup: Both servers actively handle traffic (vs Active-Passive backup)
  • Primary-Replica DB: Primary handles writes, Replica provides backup and read scaling

üóÑÔ∏è Database Cluster Explanation:

  • Primary Node: Handles all write operations (INSERT, UPDATE, DELETE)
  • Replica Node: Receives copied data from Primary, handles read operations
  • Application Difference: App writes to Primary, can read from both nodes
  • Synchronization: Primary automatically syncs data to Replica

‚ö†Ô∏è Infrastructure Issues:

  • SPOF - Load Balancer: If HAProxy fails, entire site goes down
  • SPOF - Primary Database: If Primary DB fails, no writes possible
  • Security Issues: No firewall protection against attacks
  • No HTTPS: Data transmitted in plain text (insecure)
  • No Monitoring: Cannot detect issues, performance problems, or failures
